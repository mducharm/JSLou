Example Use-Case

Financial Aid Data-Entry @ The Learning House

The problem

  • 30-40 daily calls
  • Tedious typing, lots of detailed notes after each call
  • Complex but repetitive information

The solution

CallHelper, a Vue.js app for data-entry

  • User-friendly, suitable for multi-tasking
  • Create notes by clicking, not typing
  • Customize based on your note-taking style

Some rough math:

  • 1 minute per call
  • 30 calls per day on average
  • 10~ counselors

1 minute * 30 calls * 10 counselors
=
300 minutes, or 5 hours a day

Vue.js is a progressive JavaScript framework for building user interfaces.


Vue allows for fast prototyping in a declarative style.

Declarative vs Imperative Programming

Declarative Imperative
What a program should do How a program should operate
Expresses logic of application Concerned with how statements are evaluated
More abstracted Less abstracted

In other words: you declare to Vue what you want it to do (the logic of your application), and it will handle the details involved with making this happen.

Declarative Rendering Example


                <div>
                  {{ healthyMeal }}
                </div>

                <script>
                var app = new Vue({
                  el: '#app',
                  data: {
                    healthyMeal: "donut bacon cheeseburger"
                  }
                })
                </script>

... you can do two-way binding as well:


                <div>
                    {{ healthyMeal }}
                    <input v-model="healthyMeal" >
                </div>

                <script>
                var app = new Vue({
                  el: '#app',
                  data: {
                    healthyMeal: "donut bacon cheeseburger"
                  }
                })
                </script>

... you can render data conditionally:


                <div>
                  <p v-if="healthy">
                    {{ healthyMeal }}
                  </p>
                </div>

                <script>
                var app = new Vue({
                  el: '#app',
                  data: {
                    healthy: false,
                    healthyMeal: "donut bacon cheeseburger"
                  }
                })
                </script>

... you can also render data in a loop:


                <div>
                  <p v-for="meal in meals">
                    {{ meal }}
                  </p>
                </div>

                <script>
                var app = new Vue({
                  el: '#app',
                  data: {
                      meals: [
                        "donut bacon cheeseburger", 
                        "fried Koolaid", 
                        "fried oreos"
                        ]
                  }
                })
                </script>

Vue Components

Components allow you to reuse structure, functionality, and styling throughout your application.

Example Component


                var myComponentObject = {
                  template: `
                  <div>
                    <p>Component HTML here</p>
                    {{ myComponentData }} 
                  </div>
                  `,
                  data() {
                    return {
                      myComponentData: "heya im a component"
                    }
                  },
                  methods: {
                    changeData: {
                      this.myComponentData = "change data"
                    }
                 }
                }

Registering your component 


                var vm = new Vue({
                  el: ...,
                  components: {
                    "my-component": myComponentObject
                  },
                  data: ...,
                  methods: ...,
                 })
Now this component can be used in your app as <my-component />

Single-File Components 


                <template>
                    <div>
                     ...
                    </div>
                  </template>
                  
                  <script>
                    ...
                  </script>
                  
                  <style >
                    ...
                  </style>
  • 3 parts: HTML, JavaScript, and CSS
  • Saved with .vue extension, e.g. Recipe.vue

Single-File Component Example 


                <template>
                    <div>
                      <p>Recipe:</p>
                      <p class="myRecipe">{{ recipeName }}</p>
                    </div>
                  </template>
                  
                  <script>
                  
                  export default {
                    name: "Recipe",
                    data: () => {
                      return {
                        recipeName: "Bundt Cake" 
                      };
                    }
                  };
                  </script>
                  
                  <style scoped>
                    .myRecipe {
                      color: blue 
                    }
                  </style>

... and then import your component: 


              <template>
                  <div>
                    <Recipe /> 
                  </div>
                </template>
                
                <script>
                import Recipe from "./components";
                
                export default {
                  components: {
                    Recipe
                  },
                  ...

Components can have custom properties, or "props"

Inside component:

                      export default {
                        name: "Recipe",
                        props: [
                          "name", "instructions", "ingredients"
                        ]
                        ...
                      }
Outside component:
                    
                        <Recipe v-bind:name="recipe.name"
                                   :instructions="recipe.instructions"
                                   :ingredients="recipe.ingredients"
                        />

... so when you instantiate your component, you can plug in any data you want: 


                <template>
                    <div>
                      <Recipe 
                          :name="recipe.name" 
                          :ingredients="recipe.ingredients"
                          :instruction="recipe.instructions" /> 
                    </div>
                  </template>
                  
                  <script>
                  import Recipe from "./components";
                  
                  export default {
                    components: {
                      Recipe
                    },
                    data: {
                      recipe: {
                        name: "Chicken Fried Rice",
                        ingredients: ["chicken", "rice", "broccoli"],
                        instructions: ["Make rice", "Saute chicken & broccoli", "Enjoy"]
                      }
                    }
                    ...

Data flows downstream through props.

Components can have custom events, too

Inside component:

                      export default {
                        name: "Recipe",
                        methods: => {
                          childMethod() {
                            this.$emit("favorite");
                          }
                      ...
                    }
Outside component:

                <Recipe v-on:favorite="parentMethod"
                              @favorite="parentMethod"
                /> 
            

                <template>
                    <div>
                      <p>Chicken Korma</p>
                      <button
                          v-on:click="childMethod"
                          @click="childMethod">
                          Favorite this recipe</button>

                    </div>
                  </template>
                  
                  <script>

                  export default {
                    name: "Recipe",
                    methods: => {
                      childMethod() {
                        this.$emit("favorite");
                      }
                    }
                  };
                  </script>

What it looks like from the outside:


                  <template>
                      <div>
                        <Recipe @favorite="parentMethod"/>
                      </div>
                    </template>
                    
                    <script>
                    import Recipe from "./components";
                    
                    export default {
                      components: {
                        Recipe
                      },
                      methods: => {
                        parentMethod() {
                          console.log("A recipe has been favorited.")
                        }
                      }
                    };
                    </script>

You can give arguments to $emit() to pass data to the parent component: 


                    export default {
                      name: "Recipe",
                      methods: => {
                        childMethod() {
                          this.$emit("favorite", "myFavoriteRecipe");
                        }
                    ...
                  }
                

                    <Recipe @favorite="parentMethod"/>
                    ...
                    parentMethod(e) {
                      this.favoriteRecipes.push(e);
                    }

Data flows upstream through emitted events.
